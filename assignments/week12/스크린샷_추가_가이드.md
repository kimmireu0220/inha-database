# Week 12 과제 스크린샷 추가 가이드

## 📸 스크린샷 추가 방법

### 방법 1: MySQL Workbench 사용 (권장)

#### 1단계: MySQL Workbench 실행 및 데이터베이스 연결
1. MySQL Workbench 실행
2. `WEEK11_INHA_DB` 데이터베이스에 연결
3. 새 쿼리 탭 열기

#### 2단계: 각 단계별 실행 및 캡처

**STEP 1: 데이터 개수 확인**
```sql
USE WEEK11_INHA_DB;
SELECT COUNT(*) AS total_students FROM Student;
```
- 실행 후 결과 화면 캡처
- 저장: `screenshots/01_data_count.png`

**STEP 2-1: SHOW INDEX (인덱스 적용 전)**

**STEP 3-1: 인덱스 생성**
```sql
CREATE INDEX idx_sname ON Student(Sname) USING BTREE;
CREATE INDEX idx_semail ON Student(Semail) USING BTREE;
CREATE INDEX idx_sphonenum ON Student(Sphonenum) USING BTREE;
CREATE INDEX idx_did ON Student(Did) USING BTREE;
CREATE INDEX idx_did_sname ON Student(Did, Sname) USING BTREE;
CREATE INDEX idx_studentid_email ON Student(StudentId, Semail) USING BTREE;
```
- 각 인덱스 생성 메시지 캡처
- 저장: `screenshots/08_create_indexes.png`

**STEP 3-2: SHOW INDEX (인덱스 적용 후)**
```sql
SHOW INDEX FROM Student;
```
- 실행 후 결과 화면 전체 캡처 (모든 인덱스와 Cardinality 포함)
- 저장: `screenshots/09_show_index_after.png`

#### 3단계: 스크린샷 캡처 방법

**macOS:**
- 전체 화면: `Shift + Command + 3`
- 선택 영역: `Shift + Command + 4`
- 특정 창: `Shift + Command + 4` → `Space` → 창 클릭

**Windows:**
- 전체 화면: `Windows + Print Screen`
- 선택 영역: `Windows + Shift + S`
- Snipping Tool 사용

**Linux:**
- `gnome-screenshot` 또는 `scrot` 사용

#### 4단계: 스크린샷 저장
1. `screenshots/` 폴더에 저장
2. 파일명 규칙: `01_`, `02_`, `03_` ... 순서대로
3. PNG 형식 권장

## 💡 팁

1. **스크린샷 품질**: 
   - 해상도가 충분히 높아야 텍스트가 읽기 쉬움
   - 전체 결과 테이블이 보이도록 캡처

2. **파일명 규칙**:
   - 순서대로 번호 매기기 (`01_`, `02_`, ...)
   - 설명이 포함된 이름 사용

3. **보고서 배치**:
   - 각 쿼리 실행 결과 바로 아래에 스크린샷 배치
   - 스크린샷과 분석 내용이 함께 보이도록

4. **중요 포인트**:
   - **Cardinality** 값이 명확히 보여야 함
   - **EXPLAIN** 결과의 `rows`, `filtered`, `key` 컬럼이 보여야 함
   - 인덱스 적용 전후 비교가 가능하도록 동일한 쿼리 사용
